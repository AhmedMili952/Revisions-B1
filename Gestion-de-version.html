QCM XTI103 – Git & GitHub

1. Quelle commande permet de créer une branche dev ?

git switch dev
git branch dev
git merge dev
git pull dev
Bonne réponse : git branch dev

Cette commande crée une nouvelle branche nommée dev sans changer de branche.

2. Qu’est-ce qu’une branche Git ?

Une copie complète du dépôt
Une ligne de développement indépendante
Un simple dossier
Un commit spécial
Bonne réponse : Une ligne de développement indépendante

Une branche permet de travailler sur une fonctionnalité sans impacter la branche principale.

3. Qu’est-ce qu’un commit Git ?

Une modification temporaire
Une sauvegarde de l’état du projet
Une branche distante
Un dépôt GitHub
Bonne réponse : Une sauvegarde de l’état du projet

Un commit est un instantané du projet à un moment donné, stocké dans l’historique Git.

4. Que fait la commande git merge ?

Supprime une branche
Intègre une branche dans une autre
Renomme une branche
Annule un commit
Bonne réponse : Intègre une branche dans une autre

La fusion permet de combiner les modifications d’une branche source dans la branche courante.

5. Que représente HEAD dans Git ?

Le premier commit
Le commit parent du prochain commit
Le dépôt distant
Un tag GitHub
Bonne réponse : Le commit parent du prochain commit

HEAD pointe vers le commit actuel, généralement le dernier de la branche active.
