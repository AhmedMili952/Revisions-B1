QCM XTI103 — Git & GitHub

1. Quel est l’objectif principal de la gestion de versions ?

La gestion de versions sert à suivre, retrouver et comprendre les modifications du projet.

2. Git est un système de gestion de versions :

Chaque développeur possède une copie complète du dépôt.

3. Que fait la commande git init ?

git init crée le dossier .git et démarre le versionnage.

4. Quelle commande permet de créer une branche dev ?

git branch crée une branche sans s’y déplacer.

5. Qu’est-ce qu’un commit Git ?

Un commit enregistre un snapshot du projet à un instant donné.

6. À quoi sert la staging area ?

Elle permet de choisir précisément ce qui sera commit.

7. Quelle commande ajoute un fichier au prochain commit ?

git add place les fichiers dans la staging area.

8. Que fait git merge ?

git merge combine l’historique de deux branches.

9. Que représente HEAD ?

HEAD pointe sur le commit actuellement utilisé.

10. Quelle commande récupère ET fusionne les commits distants ?

git pull = fetch + merge.
